Installation & Deployment

Before You Begin

Setting up Willow takes about 30 minutes. Once it's done, you won't need to do it again. Give yourself quiet space to move through each step without rushing.

What You'll Need

1
A Google Account
Willow runs entirely within Google's ecosystem — Sheets, Drive, Gmail, and Calendar. You'll use the Google account that holds (or will hold) your practice spreadsheet.
2
Your Practice Spreadsheet
The Google Sheet with your client data, templates, and practice records. If you're starting fresh, Willow's setup will create the structure for you.
3
An Anthropic API Key
Willow uses Claude AI to understand your requests and take action. You'll need an API key from console.anthropic.com. If you received a trial key, use that to get started.
4
The Code Files
Three files will be provided to you: EmeraldAPI (backend), EmeraldAI (AI layer), and EmeraldUI (interface). You'll paste these into your Google project.
About Your API Key

Your API key is what connects Willow to the AI that powers her. Think of it as a private passcode between your Willow and the AI service. It's stored securely in your Google project and never shared with anyone.

Trial key: If you were given a trial key, it will work for your first period. After that, you can get your own key at console.anthropic.com or ask about a monthly plan.

1

Install the Code

You'll add three files to your Google Sheet's built-in code editor. This works best on a computer.

Step 1: Open Apps Script

1
Open your practice spreadsheet
Go to your Google Sheet in a web browser (Chrome recommended).
2
Open the script editor
Click Extensions in the menu bar, then Apps Script. This opens the code editor in a new tab.

Step 2: Create the Files

In the Apps Script editor, you'll create three new files. For each one:

3
Create the API file
Click the + next to "Files" → select Script → name it EmeraldAPI. Paste the provided API code into this file.
4
Create the AI file
Click + → Script → name it EmeraldAI. Paste the provided AI code.
5
Create the UI file
Click + → HTML → name it EmeraldUI. Paste the provided interface code.
6
Save everything
Press Ctrl+S (or Cmd+S on Mac) to save all files. You should see three new files in the left sidebar alongside any existing code.
Nothing will break. Adding these files doesn't affect anything that's already working. Your existing sidebar, forms, and automations continue to run exactly as before.
2

Configure Willow

Step 3: Run First-Time Setup

7
Go back to your spreadsheet
Switch back to the Google Sheet tab. Refresh the page so the new menu items appear.
8
Run the setup
In the menu bar, click Doula Tools → Setup → Configure Settings. This creates your default configuration.
9
Create the Template Registry
Click Doula Tools → Setup → Create Template Registry. This creates the sheet that manages your document templates.

Step 4: Enter Your API Key

10
Open Project Settings
In the Apps Script editor, click the gear icon (Project Settings) in the left sidebar. Scroll down to Script Properties.
11
Add your API key
Click Add Script Property. Set the name to ANTHROPIC_API_KEY and paste your API key as the value. Click Save.

Step 5: Personalize (Optional)

While you're in Script Properties, you can also set these optional values:

Property NameWhat It DoesDefault
AI_NAMEWhat your assistant calls herselfWillow
PRACTICE_NAMEYour practice name (appears in header)Haven, The Awakening Doula
PRACTITIONER_NAMEYour name and credentialsCarlie Wyton, MA
SESSION_DURATION_MINUTESDefault calendar event length60
Change these anytime. Settings take effect immediately — just refresh Willow in your browser.
3

Deploy & Go

Step 6: Deploy as Web App

12
Create the Web App
In the Apps Script editor, click Deploy → New deployment. For type, select Web app. Set "Execute as" to Me and "Who has access" to Only myself. Click Deploy.
13
Authorize permissions
Google will ask you to authorize. Click Authorize access, choose your account, and if you see "Google hasn't verified this app," click Advanced → Go to [project name]. This is safe — it's your own code running in your own account.
14
Copy your Willow URL
After deploying, you'll see a Web App URL. Copy this link — this is your Willow's permanent home address.

Step 7: Add to Your Phone

15
Open the URL on your phone
Send yourself the Web App URL (text, email, or notes app) and open it in Safari or Chrome on your phone.
16
Add to Home Screen
iPhone: Tap the share button (square with arrow) → Add to Home Screen.
Android: Tap the three-dot menu → Add to Home Screen.

Willow now appears as an app icon on your phone. Tap it anytime to open.
That's it. You're done.

Willow is now ready. Open her from your home screen and start managing your practice. Refer to the Willow User Guide for everything she can do.

4

After Setup

Updating Willow

When you receive updated code files, repeat Steps 1–2: open Apps Script, paste the new code into the existing files (replacing the old code), and save.

If the update requires redeployment, you'll be told. In that case:

  1. Go to Deploy → Manage deployments
  2. Click the pencil icon next to your deployment
  3. Change "Version" to New version
  4. Click Deploy

Your URL stays the same — no need to re-bookmark.

Changing Your API Key

  1. Go to console.anthropic.com and generate a new key
  2. In Apps Script → Project Settings → Script Properties
  3. Update the ANTHROPIC_API_KEY value
  4. Click Save — takes effect immediately

Wiring Templates

Willow generates documents from Google Doc templates. You'll want to connect your templates to the system:

  1. Go to Doula Tools → Manage Templates
  2. Search for your template by name (searches your Google Drive)
  3. Click Wire to connect it

Or just tell Willow: "Wire my Session Notes template" and she'll walk you through it.

If Something Goes Wrong

Willow says "Something went wrong"

Check that your API key is set correctly in Script Properties. Make sure there are no extra spaces before or after the key.

Clients aren't loading

Refresh the page. If it persists, open Apps Script and check for any error messages in the Execution Log (View → Executions).

A button is greyed out

The template for that action hasn't been wired yet. Go to Doula Tools → Manage Templates to connect it.

"Google hasn't verified this app"

This is normal for personal projects. Click Advanced → Go to [project name]. It's your own code running in your own account.
